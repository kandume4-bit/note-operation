# article-template.md — note記事の雛形

記事を生成するときの骨格。**株式・快眠の共通骨格**＋それぞれの差分を下に置く。
すべて `persona.md`（文体・共通点・親しみやすさ）と `title-rules.md`（タイトル）に従う。

---

## 共通骨格（両ジャンル共通）

```
【タイトル】
　title-rules.md に従い最低5案→採用1案。数字＋ベネフィット＋具体性。

【冒頭3行（つかみ）】
　読者の悩みをズバリ言い当てる。「それ、私だ」と思わせる。
　例：「寝ても昼間眠い」「投資、始めたいけど損が怖い」

【共感・共通点】
　「私も同じでした」を1つ。失敗や悩みを先に打ち明けて距離を縮める。
　（persona.md：読者との共通点・親しみやすさ）

【本文：1記事1メッセージ】
　言いたいことは1つだけ。仕組み→具体例（数字・固有名詞）→どうすればいいか。
　専門用語は生活の言葉で言い換えてから使う。

【今日のアクション／持ち帰り】
　読者が今日（今夜）できる具体行動を1つ。数字で示す。

【書き手の一言（締め）】
　等身大の本音。押し付けず、伴走する。投資は「最後はご自身で判断を」。

【連載リンク（快眠のみ）】
　← 前回Day ／ 次回Day →
```

仕上げチェック（投稿前）：
- [ ] 1記事1メッセージになっているか
- [ ] 数字・具体（固有名詞・体験）が入っているか
- [ ] 読者との共通点・親しみやすさが1つ以上あるか
- [ ] 冒頭3行とタイトルが矛盾していないか（釣りでない）
- [ ] （投資）断定していないか／（健康）受診の目安を添えたか

## 株式記事 差分

```
【Day/曜日テーマ】
　stock-knowledge.md の曜日テーマ＋当日の鮮度ネタ。

【最新相場の取り込み】（鮮度ルール）
　・当日終値：日経平均・TOPIX（前日比%／2026-MM-DD終値）
　・米国市場（前日）・ドル円・金利
　・資金が集まったセクター／当日の主要ニュース
　→ 生データの羅列で終わらせず「初心者はどう捉えるか」の解釈を付ける。

【本文の型例】
　今日の相場で何が起きたか → なぜか（かみ砕く） → 初心者がどう考えるか →
　今日の学び（1つ）。特定銘柄の推奨はしない。考え方を渡す。

【締め】
　「あくまで私の一例です。最後はご自身で判断を。」
```

保存先：`articles/stock/YYYY-MM-DD.md`

## 快眠記事 差分

```
【バッジ】
　Day○/14｜（その日の一言テーマ）   ※Day14後は「応用編Day○」等

【冒頭3行】
　その日のテーマに沿った悩みを言い当てる。

【なぜそうなるか】
　仕組みを1つだけ、かみ砕いて。

【今夜のアクション】
　今晩すぐできる具体行動を1つ（数字付き：15分／90分前 など）。

【書き手の一言】
　運転中の睡魔・サプリ等の実体験を等身大で。

【連載リンク】
　← Day(n-1)へ ／ 次回 Day(n+1)へ →   （URLは published.md から補完）

【免責】
　強い不眠・無呼吸の疑いは医療機関へ。効果には個人差があります。
```

保存先：`articles/sleep/dayNN.md`（例 day03.md）
