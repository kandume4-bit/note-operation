# threads-template.md — Threads投稿の雛形

note記事2本を紹介する短文を生成する。Threadsは**1投稿500文字**まで。
役割：noteへの導線。**続きが気になって押したくなる**ひとことを作る。

---

## 基本ルール

- **2記事それぞれに1本**ずつ短文を作る（株式・快眠）。
- 文体は `persona.md` に従う（敬体＋たまに本音／親しみやすさ／煽らない）。
- **冒頭1行で引きを作る**（悩み・意外・数字）。タイトルのコピペで終わらせない。
- **リンクは末尾**に1つ。「続きはnoteで↓」など自然な導線。
- ハッシュタグは2〜4個まで（つけすぎない）。
- 数字・具体を1つは入れる。釣りで終わらせない（noteの中身と一致）。

---

## テンプレA：株式記事用

```
【今日の相場メモ】
（当日の値動き or 気づきを1行。例：今日の日経は◯◯円、△△が買われました）

（初心者目線の一言。例：こういう日、私はこう考えています）

詳しくはnoteに書きました↓
（note URL）

#新NISA #高配当株 #投資初心者
```

---

## テンプレB：快眠記事用

```
Day○/14｜（テーマの一言）

（悩みの言い当て1行。例：「寝てるのに昼間眠い」あの感じ、ありますよね）

今夜できることを1つだけ書きました。
（具体アクションを匂わせる。例：ポイントは“朝の15分”です）

続きはnoteで↓
（note URL）

#睡眠の質 #快眠 #働く世代
```

---

## 生成フォーマット（毎回この形で2本出す）

```
■ Threads（株式）
〜〜〜〜（本文）
note URL（投稿後に貼る）

■ Threads（快眠）
〜〜〜〜（本文）
note URL（投稿後に貼る）
```

> URLは投稿後に確定するため、生成時は「（ここにnote URL）」のプレースホルダで出す。
> 投稿後、`logs/published.md` の「Threads済」を○にする。

---

## 投稿フロー（予約投稿ツールで時間差＋プレビューカード）

Threadsへの投稿は**予約投稿ツール**を使う運用（ユーザー決定）。

1. noteに記事を投稿し、**note URL**を取得する。
2. 生成済みのThreads短文の「（ここにnote URL）」を実URLに置き換える。
3. **Meta Business Suite**（無料・Threads対応）などの予約投稿ツールに貼り、**時間差**で予約する。
   - 代替：Threadsアプリ内の予約機能、Buffer / Hootsuite / Later など。
4. **プレビューカードは自動**。本文にURLが入っていれば、Thread側でリンクの画像つきカードが生成される（こちらで画像添付は不要）。
5. 予約完了後、`logs/published.md` の「Threads済」を○にする。

時間差の目安：note投稿の**30分〜数時間後**、または翌朝の通勤時間帯など、読まれやすい時間に予約すると効果的。同時より少しずらすほうが両方の露出を取りやすい。

> 注意：noteには投稿APIがなく、Threadsの自動投稿コネクタも未接続のため、「note投稿を検知して全自動でThreads投稿」は現状不可。URLの差し込みと予約だけ手動で行う半自動運用。
