# threads-template.md — Threads投稿の雛形

note記事2本を紹介する短文を生成する。Threadsは**1投稿500文字**まで。
役割：noteへの導線。**続きが気になって押したくなる**ひとことを作る。

---

## 基本ルール

- **2記事それぞれに1本**ずつ短文を作る（株式・快眠）。
- 文体は `persona.md` に従う（敬体＋たまに本音／親しみやすさ／煽らない）。
- **冒頭1行で引きを作る**（悩み・意外・数字）。タイトルのコピペで終わらせない。
- **リンクは末尾**に1つ。「続きはnoteで↓」など自然な導線。
- ハッシュタグは2〜4個まで（つけすぎない）。
- 数字・具体を1つは入れる。釣りで終わらせない（noteの中身と一致）。

---

## テンプレA：株式記事用

```
【今日の相場メモ】
（当日の値動き or 気づきを1行。例：今日の日経は◯◯円、△△が買われました）

（初心者目線の一言。例：こういう日、私はこう考えています）

詳しくはnoteに書きました↓
🔗（note URL）

#新NISA #高配当株 #投資初心者
```

---

## テンプレB：快眠記事用

```
Day○/14｜（テーマの一言）

（悩みの言い当て1行。例：「寝てるのに昼間眠い」あの感じ、ありますよね）

今夜できることを1つだけ書きました。
（具体アクションを匂わせる。例：ポイントは“朝の15分”です）

続きはnoteで↓
🔗（note URL）

#睡眠の質 #快眠 #働く世代
```

---

## 生成フォーマット（毎回この形で2本出す）

```
■ Threads（株式）
〜〜〜〜（本文）
🔗 note URL（投稿後に貼る）

■ Threads（快眠）
〜〜〜〜（本文）
🔗 note URL（投稿後に貼る）
```

> URLは投稿後に確定するため、生成時は「🔗（ここにnote URL）」のプレースホルダで出す。
> 投稿後、`logs/published.md` の「Threads済」を○にする。
